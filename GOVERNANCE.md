# Governance

This project is maintained by the cloud-itonami team as part of the Open Occupation Blueprints
initiative. Decisions about the project are guided by the principles documented in this file.

## Decision Making

- **Bug fixes and minor improvements** are merged by maintainers with code review.
- **Major features or scope changes** require discussion in an issue before a PR is opened.
- **Scope boundaries** (especially restrictions on military/sensitive operations) are permanent
  and cannot be changed without explicit consensus from maintainers and stakeholders.

## Scope Boundaries

This actor is explicitly designed to support **administrative operations only**.
Any proposal to add operational command, weapons systems, targeting, or lethal autonomous
capability is **permanently out of scope** and will be rejected.

See the README's "SCOPE BOUNDARIES" section for details.
