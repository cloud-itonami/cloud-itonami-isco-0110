# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please email the maintainers at
the email address listed on the project GitHub page. Do NOT open a public issue.

Please include:

1. A description of the vulnerability
2. Steps to reproduce (if applicable)
3. Potential impact
4. Suggested fix (if you have one)

We will acknowledge receipt of your report within 48 hours and will work to resolve
the issue in a timely manner.

## Security Scope

This project is explicitly scoped to administrative operations only. Any security
issue related to attempted expansion of scope (e.g., circumventing the scope boundaries
defined in the README) is a critical security issue and should be reported immediately.

## Supported Versions

- Latest version (main branch) receives security updates.
- Previous versions receive critical security fixes for 12 months after release.
